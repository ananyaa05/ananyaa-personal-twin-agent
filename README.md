# LPI Digital Twin Architecture Auditor
An Explainable AI (XAI) agent built for the LPI Developer Kit Level 3. This agent acts as a Senior Systems Architect, taking a user's digital twin concept and cross-referencing it against the official SMILE methodology to generate a "Missing Reality Report".

## Key Engineering Features
1. **Inter-Process Communication (IPC):** Bypasses local pathing errors by using relative directory referencing to establish a live `subprocess` connection directly to the LPI Node.js MCP Server.
2. **Strict Data Provenance:** Implements a multi-tool pipeline where every critique generated by the LLM must be explicitly hard-traced to a specific LPI database query.
3. **Local Execution:** Powered entirely offline by Ollama to ensure data privacy and zero-cost processing.

## LPI Tools Integrated
The agent natively orchestrates the following LPI tools via standard JSON-RPC standard I/O:
* `smile-overview`: Retrieves baseline framework and safety hazards.
* `query-knowledge`: Searches for methodology best practices.
* `get-case-studies`: Retrieves past failure metrics.
* `smile-phase-detail`: Audits specific emulation gaps.

## Tech Stack
* **Language:** Python 3.x
* **Brain:** TinyLlama (via Ollama)
* **Architecture:** Tool-Augmented Generation with Native Node.js MCP Integration

## How to Run Locally
1. Ensure the `lpi-developer-kit` repository is located in the same parent directory as this agent repository.
2. Ensure **Ollama** is installed and running (`ollama serve`).
3. Clone this repository.
4. Run the agent by passing your concept as a command-line argument:
   ```bash
   python agent.py "I want to build a digital twin of Amity University's CSE lab"
    ```
